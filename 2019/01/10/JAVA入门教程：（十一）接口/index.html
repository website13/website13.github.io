Java入门教程：（十一）接口

|阅读量:

接口是一种用interface声明的类，里面只定义方法，没有方法体，需要由另一个类去用关键字implements实现接口里的全部的方法。

上面的代码就是一个完整的从定义到使用的过程，它的功能是计算两数相乘。有人会问为什么要用接口，直接使用ShuXueImplA和ShuXueImplB不行吗，非得先实现接口再通过接口调用方法？你可以把接口看作一个标准，他只提供方法名，不实现过程。就跟你领导同时让小张和小明整理同一摞纸质文档一样，“整理文档”这件事相当于接口，小张和小明相当于两个实现类，小张可能先分类再整理，把文档按一定规律贴上标签，小明可能直接整理，结果都是一样整整齐齐一分不少，但是小张整理的纸质文档因为有标签所以查询的时候快，但是往里面放入新的文档时也需要分类，所以插入慢，小明整理的纸质文当因为整理的时候没做标签所以查询慢但是以后有大量的文档加入时就快。这两人不同的整理过程相当于实现类的方法。领导可以根据实际需求来让他俩其中一个人干活。

上节教程中讲的集合，实际情况我们不会直接写这样的代码“ArrayList arrayList = new ArrayList();”，因为List是集合接口，ArrayList、Vector、LinkedList都是实现了集合接口的类，所以我们使用的时候应该这样声明

上节还说了HashMap，它实现了Map接口，HashSet，它实现了Set接口。使用他们时的时声明的方式与上面List一样，前面写接口名，后面new实现类名，具体用哪个实现类需要根据自己的需求去选择。
